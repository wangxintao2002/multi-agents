# 基于大语言模型的多智能体系统（LLM-MAS）调研报告

> 调研范围：近两年（2023 下半年–2025）面向 LLM-based Multi-Agent Systems（LLM-MAS）的学术论文、综述与基准。
> 调研方法：多角度并行检索 → 抓取 22 篇一手来源 → 抽取 100 条断言 → 对其中 25 条核心断言做 3 票对抗式核验（需 2/3 反驳才否决）→ 综合。
> 报告日期：2026-06-04。

---

## 摘要（TL;DR）

基于大语言模型的多智能体系统（LLM-MAS）的兴起，根本动因是**单个 LLM 存在固有局限**——幻觉、自回归本质（难以"慢思考"）、以及 scaling law 的边际递减。研究者由此转向**横向扩展（horizontal scaling）**：让多个拥有专业角色与技能的智能体协作，以获得超越个体之和的**集体智能（collective intelligence）**，部分综述甚至将其框定为通往通用人工智能（AGI）的一条可能路径。

近两年的工作可归纳为五条主线：
1. **角色分工与编排**——MetaGPT（SOP+流水线）、CAMEL（role-playing + inception prompting）、AutoGen（可对话智能体 + conversation programming）。
2. **协作与辩论机制**——多智能体辩论（MAD）、ChatEval 用于推理与评估。
3. **通信结构与协议**——从合作/辩论/竞争三范式，到 MCP/ACP/A2A/ANP 等互操作协议。
4. **可扩展性**——全连接拓扑导致组合爆炸与隐私泄露；MegaAgent 尝试扩展到 590 个智能体并去除预定义 SOP。
5. **失败模式与可靠性**——MAST 失败分类法系统性地揭示了"为什么多智能体系统会失败"。

**一个贯穿全篇的关键警示**：多智能体相对单智能体的"净收益"并非无条件成立。多智能体辩论提升推理的结论在后续受控研究中存在争议，收益可能更多来自模型多样性/强度而非辩论机制本身；"辩论提升事实性、减少幻觉"以及"任务分解+协作必然提升性能"等笼统断言在核验中被否决。该领域目前正从"能不能搭起来"转向"为什么不可靠、如何评估"。

## 第一部分：这个领域要解决什么问题？

### 1.1 为什么需要多智能体，而非单智能体？

核验等级 **high**（两份独立一手综述一致，3-0 通过）。

驱动力来自单个 LLM 的三类固有局限（Tran et al., 2501.06322 明确列举）：

- **幻觉（hallucination）**：单一模型的输出缺乏外部制衡，错误会被直接采纳。
- **自回归本质（auto-regressive nature）**：逐 token 生成、难以进行"慢思考"式的反复审议与回溯。
- **Scaling law 的边际**：单纯把模型做大，性价比与能力增益递减。

对应的解法是**横向扩展**：与其把一个模型做得更大，不如让多个具备**多样化角色画像（diverse profiles）与专业技能**的智能体通过**相互交互**和**集体决策**来求解（Guo et al., 2402.01680：MAS 旨在"leverage the collective intelligence and specialized profiles and skills of multiple agents"）。

> ⚠️ **核验否决项**：把动因表述为"多智能体通过任务分解+协作**必然**提升性能"是**不准确**的——该笼统断言在 Chen et al.(2412.17481) 上被 0-3 反驳。正确表述应是"追求集体智能、克服单体局限"，而**性能提升是有条件的、需要逐任务验证的**。

### 1.2 领域被如何"框定"

- Li & Wang 等综述将 LLM-MAS 框定为**"通往通用人工智能的一条有前景的路径"**（注意：这是综述作者的框定/愿景，而非客观已证事实），并将应用分为**问题求解（problem-solving）**与**世界仿真（world simulation）**两大领域。
- Chen et al.(2412.17481) 从应用目的出发，归纳为三类：**任务求解、特定场景仿真、生成式智能体评估**；该综述系统收集了 ACL/NeurIPS/AAAI/ICLR 2023–2024 顶会的 **125 篇论文**（顶会子集）。

## 第二部分：综述全景与分类维度

近两年出现了多份系统性综述，各自给出了不同但互补的"坐标系"。

### 2.1 Guo et al., 2402.01680（IJCAI 2024）——四维组织框架

核验 **high（3-0）**。沿四个关键维度组织整个领域：

| 维度 | 子类 |
|---|---|
| 智能体-环境接口（agents-environment interface） | Sandbox / Physical / None |
| 智能体画像（agent profiling） | Pre-defined / Model-Generated / Data-Derived |
| 智能体通信（agent communication） | 见下 §2.4 |
| 能力获取（capability acquisition） | Memory / Self-Evolution / Dynamic Generation |

### 2.2 Tran et al., 2501.06322（2025-01）——协作机制五维框架

核验 **high（3-0）**。提出可扩展的协作刻画框架：

- **actors**：参与的智能体
- **types**：合作（cooperation）/ 竞争（competition）/ 竞合（coopetition）
- **structures**：点对点（peer-to-peer）/ 中心化（centralized）/ 分布式（distributed）
- **strategies**：基于角色（role-based）/ 基于模型（model-based）
- **coordination protocols**：协调协议

### 2.3 Li & Wang 等（Springer 2024, DOI 10.1007/s44336-024-00009-2）——统一五组件结构

核验 **high（3-0）**。把以往工作合成为统一结构：**profile（画像）/ perception（感知）/ self-action（自我行动）/ mutual interaction（相互交互）/ evolution（演化）**。

### 2.4 通信的系统化分类

核验 **high（3-0）**。综述普遍把"通信"拆开来分析：

- **通信范式**：合作（Cooperative）/ 辩论（Debate）/ 竞争（Competitive）——多份综述高度一致。
- **通信结构**：分层（Layered，如 DyLAN）/ 去中心化（Decentralized）/ 中心化（Centralized）/ 共享消息池（Shared Message Pool，源自 MetaGPT）；另有综述命名为去中心化/中心化/分层/嵌套（nested）。
- **通信内容**：部分综述（Li & Wang）进一步把通信拆为"范式 / 结构 / 内容"三部分。

## 第三部分：代表性框架与系统

### 3.1 MetaGPT（2308.00352，ICLR 2024 oral）——SOP + 流水线编排

核验 **high（3-0）**。

- **问题**："朴素地串联 LLM"会因**级联幻觉（cascading hallucinations）**产生逻辑不一致。
- **方法**：把人类的**标准作业程序（SOP）编码进 prompt 序列**，采用**流水线（assembly-line）范式**为不同智能体分配专业角色（Product Manager / Architect / Engineer / QA 等），把复杂任务拆解为子任务协同完成。
- **意义**：是 role-based 编排的核心范式，也是"结构化分工优于无序串联"的有力论据。MetaGPT+GPT-4 在 HumanEval 上达 85.9% Pass@1。

### 3.2 CAMEL（2303.17760，NeurIPS 2023）——角色扮演与 inception prompting

核验 **high（3-0）**。

- **问题**：基于聊天的 LLM 的成功"严重依赖人类输入来引导对话"，既费时又难规模化。
- **方法**：提出 **role-playing** 通信智能体框架，借助 **inception prompting** 引导智能体自主完成任务，同时保持与人类意图的一致性，**减少对人工逐轮引导的依赖**。

### 3.3 AutoGen（2308.08155，COLM 2024，Microsoft）——可对话智能体 + 对话编程

核验 **high（3-0）**。

- **核心设计**：开源框架，通过多个**可定制、可对话（conversable）**的智能体相互对话来完成任务，把"多智能体对话"确立为核心编程范式。
- **conversation programming**：可用**自然语言与代码**共同编排灵活的交互模式；智能体可组合 LLM、人类输入与工具，支持灵活角色配置与 **human-in-the-loop** 协作（ConversableAgent / AssistantAgent / UserProxyAgent、`human_input_mode` 等）。

> ⚠️ **版本警示**：以上描述对应 2023 年的 **v0.2** 设计。AutoGen 在 2024–2025 已大幅重构（v0.4，引入异步事件驱动架构）。涉及实现细节时需注明版本。

### 3.4 MegaAgent（2408.09955，ACL 2025 Findings）——大规模与去 SOP

核验 **high（3-0）**，但**附强保留意见**。

- **主张**：在国家级政策仿真任务中扩展到**约 590 个智能体**，在 **2991 秒**内完成；而基线框架（AutoGen / CAMEL / AgentVerse / MetaGPT）**只能协调不到 10 个智能体**且无法产出预期结果。
- **创新点**：**去除对预定义 SOP 的依赖**——作者指出"预定义 SOP 在整个交互中保持不变"是先前框架的关键局限。

> ⚠️ **可靠性警示**：590 智能体 / 2991 秒 / 基线<10 均为**作者单次实验自报数据，未经第三方独立复现**。其"O(log n) 通信开销"的复杂度断言在核验中被 **0-3 反驳，不可引用**。

## 第四部分：协作与辩论机制（推理与评估）

### 4.1 多智能体辩论 MAD（Du et al., 2305.14325，ICML 2024）

核验 **medium（推理收益 2-1，有争议）**。

- **方法**：多个 LLM 实例在多轮中各自提出并**辩论**推理过程，再收敛到共识答案。
- **原论文报告**：显著提升数学与策略推理。

### 4.2 ChatEval（2308.07201，ICLR 2024）

核验 **high（3-0）**。把多智能体辩论用于**评估**：构建"多智能体裁判团"自主讨论并评估生成文本质量，较单智能体评估提升可达约 6.2%。

### 4.3 ⚠️ 关键争议：辩论的收益是真的吗？

这是整篇报告**最重要的可靠性警示**：

- **被否决的断言（0-3）**："多智能体辩论提升事实有效性、减少谬误答案与幻觉。"——该笼统断言未通过核验。
- **后续受控研究的质疑**（2511.07784、2511.11306、2504.05047 等）指出：辩论带来的收益**可能更多源于参与模型的多样性/强度，而非辩论机制本身**，且**并非总能复现**。

因此，MAD 的推理收益应表述为"**原论文在其特定基准上的报告**"，而非领域定论。在受控、可复现设置下，多智能体相对单智能体的净收益**在何种任务边界内成立、有多大**，仍是开放问题（见 §7）。

## 第五部分：通信协议与互操作（基础设施层）

随着智能体生态扩张，"智能体之间如何发现彼此、委派任务、共享上下文、维持安全"成为新焦点。Ehtesham et al.（2505.02279）综述了四种新兴互操作协议：

| 协议 | 定位 | 关键机制 |
|---|---|---|
| **MCP**（Model Context Protocol） | 工具访问 | JSON-RPC 客户端-服务端接口，安全工具调用与类型化数据交换 |
| **ACP**（Agent Communication Protocol） | 结构化多模态消息 | 基于 RESTful HTTP，MIME 类型多部分消息、同步/异步交互、会话管理、DID |
| **A2A**（Agent-to-Agent Protocol） | 协作任务执行 | 基于"能力卡（Agent Cards）"的点对点任务委派，面向企业工作流 |
| **ANP**（Agent Network Protocol） | 去中心化智能体市场 | W3C DID + JSON-LD 图实现开放网络的智能体发现 |

该综述提出分层渐进的采用路线：MCP（工具）→ ACP（多模态消息）→ A2A（协作执行）→ ANP（去中心化市场）。这是一篇概念性/比较性综述，不含新基准或定量实验。

## 第六部分：可扩展性挑战

核验 **high（3-0）**。

- **核心瓶颈**：在 LLM-MAS 中，**全连接（fully-connected）通信会导致组合爆炸（combinatorial explosion）与隐私泄露（privacy disclosure）**（Chen et al., 2412.17481, §3.1）。
- **应对方向**：转向**稀疏/分层/优化拓扑**；后续工作把"通信拓扑结构学习"列为研究优先项（2505.22467），并探索**动态生成通信拓扑**（2510.07799）。
- **隐私维度**：智能体间通信的隐私泄露成为独立风险（如 AgentLeak 类工作）。

## 第七部分：失败模式与可靠性（MAST）

> 这是该领域近期最重要的"清醒剂"之一：在大量框架涌现后，研究界开始系统追问**"为什么多智能体系统会失败"**。

**"Why Do Multi-Agent LLM Systems Fail?"（2503.13657）** 提出了首个**多智能体系统失败分类法 MAST（Multi-Agent System Failure Taxonomy）**：

- **数据**：MAST-Data，收集自 **7 个主流 MAS 框架**的 **1600+ 条标注 trace**（声称是首个系统刻画 MAS 失败动态的数据集）。
- **方法**：由专家人工标注 150 条 trace 构建分类法，标注者间一致性 **kappa = 0.88**；并配套 **LLM-as-a-Judge** 流水线实现可扩展自动标注，跨 GPT-4 / Claude 3 / Qwen2.5 / CodeLlama 与编码/数学/通用任务验证。
- **分类法**：识别出 **14 种失败模式**，归为 **3 大类**：
  1. **系统设计问题（system design issues）**
  2. **智能体间错位（inter-agent misalignment）**
  3. **任务验证（task verification）**
- **核心结论**：这些失败"需要更复杂的解决方案"——**简单修补不足以解决**，指向了一条系统化的研究路线图。资源已开源（CC BY-NC-ND 4.0）。

**启示**：MAST 与 §4.3 的辩论争议相互印证——多智能体的可靠性并非自动获得，许多失败源于编排设计与智能体间协调（错位），而非单纯的模型能力不足。

## 第八部分：评估与基准

**"A Survey on Evaluation of LLM-based Agents"（2503.16416，ACL Findings 2025）** 是首份面向 LLM 智能体评估的综合综述，从五个视角组织：

1. **核心 LLM 能力**（规划、工具使用等智能体工作流所需基础技能）
2. **应用专属基准**（Web 智能体、SWE 软件工程智能体等）
3. **通用智能体（generalist agents）评估**
4. **基准维度**（智能体基准的核心维度分析）
5. **评估框架与工具**

该综述指出趋势是走向**更真实、更有挑战性、可持续更新的基准**，并点名当前关键缺口：**成本效率（cost-efficiency）、安全（safety）、鲁棒性（robustness）评估，以及细粒度、可扩展的评估方法**。

## 第九部分：开放问题与研究空白

基于本次核验，以下问题证据不足或存在分歧，是值得后续深入的方向：

1. **净收益的边界**：在受控、可复现设置下，多智能体（辩论/协作）相对单智能体的净收益究竟有多大、在何种任务边界内成立？现有证据呈现"支持（Du et al./ChatEval）"与"质疑（2511.07784 等）"并存，**缺乏权威 meta 分析定论**。
2. **失败模式的标准化**：MAST 提供了首个分类法，但失败模式的**公认标准与自动诊断/自修复**仍不成熟。
3. **记忆与长程规划**：多智能体设置下的记忆/上下文管理、长程规划与推理如何组织，本次调研证据未充分覆盖，需专门调研。
4. **安全与对齐**：智能体间通信的隐私泄露已被提及，但**对齐、越狱、共谋（collusion）风险的系统性综述仍欠缺**。
5. **本次未充分覆盖的专题**：ChatDev、AgentVerse（仅作为 MegaAgent 基线出现）、MASLab 等在本批已确认证据中证据不足，下一轮可定向补充。
